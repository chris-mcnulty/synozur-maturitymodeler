## 1) Product Overview & Principles

**Goal**\
A branded, extensible web app to host **multiple maturity models**—each at `/model-slug`—where users login, complete assessments, and receive gated, benchmarked **PDF results** by email. Admins can manage models via **CSV**, review results, and export data.

**Experience principles (Synozur voice)**\
Empathetic, people‑first, and business‑outcome focused; clear, conversational copy at a 12th‑grade level; mobile‑first, fast, and accessible.

**Brand/Legal anchors**\
Use Synozur colors, logos, and typography; include ownership and trademarks in footer and PDFs; link to the **Privacy** page from every screen. ## 2) User Journeys & Routing

**Public**

*   `/` — Landing: what the platform is, featured models, “Start” CTA
*   `/:modelSlug` — Model overview: description, opening graphic (optional), Start

## 4) Data Model (ERD & Entities)

**Entities**

*   **User**: `id, email (unique), username (unique), password_hash, name, company, job_title, industry, country, role (user|analyst|admin), created_at`
*   **Model**: `id, slug (unique), name, description, status (draft|published|archived), default_theme JSON, created_by, created_at`
*   **ModelVersion**: `id, model_id, version (semver), is_current, scale_min=100, scale_max=500, created_at`
*   **Dimension**: `id, model_version_id, dimension_key, label, weight (nullable), order`
*   **Question**: `id, model_version_id, dimension_id, question_key, text, type (mc|likert), order, required bool`
*   **AnswerOption**: `id, question_id, key, label, score (100–500), order`
*   **PreSurveyQuestion**: `id, model_version_id, key, text, type (mc|likert), order`
*   **Assets**: `id, model_version_id, opening_graphic_url, closing_graphic_url, resources JSON[{title,url}], cta JSON{label,url}`
*   **AssessmentSession**: `id, user_id, model_version_id, started_at, completed_at, profile_complete bool`
*   **Response**: `id, session_id, question_id, answer_option_id, answered_at`
*   **PreSurveyResponse**: `id, session_id, presurvey_question_id, answer_option_id, answered_at`
*   **Scorecard**: `id, session_id, overall_score, per_dimension JSON{dimKey:score}, label, benchmark_snapshot JSON, pdf_url`
*   **BenchmarkAggregate**: `id, model_version_id, industry, country, n, mean_overall, mean_per_dimension JSON, last_updated`
*   **AuditLog**: `id, actor_user_id, action, target_type, target_id, metadata JSON, created_at`

> **Notes**\
> • All scores and validation aligned to the **100–500 default** scale.\
> • Future: `Dimension.weight` supports custom weighting.\
> • Benchmarks require **n ≥ 30** per industry to display.

## 5) Scoring, Labels, and Benchmarking

**Per‑question**: AnswerOption has a score in {100,200,300,400,500}.\
**Per‑dimension**: Mean of its question scores (rounded to nearest 5).\
**Overall**: Mean of dimension scores (weights later).\
**Labels**

*   **100–199**: Nascent
*   **200–299**: Experimental
*   **300–399**: Operational
*   **400–449**: Strategic
*   **450–500**: Transformational

**Benchmarking**

*   Aggregates computed per **industry** (and optionally country) nightly or on‑demand.
*   Only shown when `n ≥ 30` to protect privacy.
*   Display user vs. industry mean (overall and per‑dimension) with deltas.

> **Model content alignment:** Based on the AI Maturity model’s dimensions and 100–500 scoring you provided; this platform generalizes those patterns across models. [3](https://www.synozur.com/)

**Validation Rules**

*   Scores must be within `[scale_min, scale_max]` (default 100–500).
*   Unique `model_slug + version` on import.
*   Required fields present; unknown `question_type` rejected (only `mc|likert`).

## 9) API (JSON over HTTPS)

**Auth**

    POST /api/auth/signup        { username, email, password }
    POST /api/auth/signin        { usernameOrEmail, password }
    POST /api/auth/forgot        { email }
    GET  /api/me                 -> current profile
    PUT  /api/me                 { name, company, job_title, industry, country }

**Models**

    GET  /api/models                     -> published models
    GET  /api/models/:slug               -> model overview + current version
    GET  /api/models/:slug/version/current

**Sessions & Results**

    POST /api/sessions                   { modelSlug } -> { sessionId }
    PUT  /api/sessions/:id               { responses[] }   # autosave
    POST /api/sessions/:id/complete      # compute scores, create Scorecard, queue PDF
    GET  /api/results/my                 -> my results
    GET  /api/results/:id                -> result + scorecard + benchmarks snapshot
    GET  /api/results/:id/pdf            -> signed URL (auth‑protected)

**Admin/Analyst**

    GET  /api/admin/summary              (analyst|admin)
    POST /api/admin/models/import-csv    (admin) multipart
    GET  /api/admin/models/export-csv?model=:slug (analyst|admin)
    PUT  /api/admin/models/:id           (admin)  # assets, presurvey, status
    POST /api/admin/assets/upload        (admin)  # images to Replit storage
    GET  /api/admin/results/export?...   (analyst|admin) # filters
    GET  /api/admin/benchmarks           (analyst|admin)
    PUT  /api/admin/benchmarks/rebuild   (admin)
    GET  /api/admin/audit                (admin)

**Responses**\
All endpoints return `{ ok: boolean, data|error }`. Validation errors include field‑level messages.

## 15) Replit Architecture & Setup

**Stack**

*   **Frontend**: Vite + React + TypeScript; React Router; Zustand; Tailwind/CSS Modules; Recharts
*   **Backend**: Node.js + Express (TypeScript); Prisma ORM → **PostgreSQL**
*   **Storage**: Replit content storage (`/uploads`) for images/PDFs
*   **Email**: **SendGrid**
*   **Auth**: Username + password (`bcrypt`), JWT (access/refresh)

**Monorepo**

    /server   # Express + Prisma
    /frontend # Vite + React
    /uploads  # persisted assets & PDFs
    .replit
    replit.nix

**.replit**

```ini
run = "npm --workspace server run dev"
```

**Environment variables**

```bash
DATABASE_URL=postgres://user:pass@host:5432/db
JWT_SECRET=replace-with-long-random
SENDGRID_API_KEY=sg.xxxxx
APP_URL=https://models.synozur.com
ASSET_STORAGE_DIR=/home/runner/<repl>/uploads
EMAIL_FROM=no-reply@models.synozur.com
```

**Prisma migration**\
Create all entities in §4; add indexes on `user.email`, `model.slug`, `model_version.is_current`.

## 16) Security Hardening

*   Passwords: `bcrypt` with strong cost factor
*   JWT: httpOnly refresh cookies; short‑lived access tokens; CSRF for state‑changing routes
*   Rate limiting on auth and import/export routes; CAPTCHA on signup
*   Input validation with Zod; strict CORS; MIME/type checks on uploads
*   Daily encrypted backups; role‑based access control; **AuditLog** for admin actions
