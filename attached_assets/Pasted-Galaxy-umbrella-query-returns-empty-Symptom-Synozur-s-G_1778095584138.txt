Galaxy umbrella query returns empty
Symptom
Synozur's Galaxy portal calls Nebula's umbrella endpoints with the issued umbrella key and a ?domain= parameter:

GET {NEBULA_BASE_URL}/api/galaxy/reports?domain=synozur.com&limit=20
GET {NEBULA_BASE_URL}/api/galaxy/workspaces?domain=synozur.com

Both return HTTP 200 with empty data:

{ "data": [], "page": 1, "limit": 20, "total": 0 }
{ "data": [] }

We confirmed:

The umbrella API key authenticates fine (no 401/403).
We tried domain=synozur.com, synozur.io, chrismcnulty.com, mcnulty.com, test.com — all return total: 0.
A request with no domain param returns {"error":"domain_required"}, so the parameter is being read.
The user reports that Nebula already has many workspaces defined, so this is almost certainly a filter / scoping issue on the Nebula side, not "no data."

Please diagnose and report back
Run these against the Nebula DB and paste the results:

1. Are there any workspaces at all, and what state are they in?

GROUP BY status
ORDER BY n DESC;

2. Would the workspaces directory match anything for synozur.com?
Whatever field your ?domain= filter uses on /api/galaxy/workspaces, run the equivalent of:

-- adjust column / join to whatever your filter actually checks
SELECT COUNT(*) AS workspaces_for_synozur

4. Verify the umbrella key's scope
In your umbrella-keys / api-keys table, dump the row for the key Synozur is using (mask the secret — we just need scope/tenant fields):

If allowed_domains is non-empty and doesn't contain synozur.com, that's the bug — the key is scoped narrower than we expect.

5. Show us the exact filter your endpoint applies
In /api/galaxy/workspaces and /api/galaxy/reports, paste the few lines where the domain query param turns into a SQL/ORM filter. Specifically we need to know:

Is domain matched case-insensitively?
Does it match against owner email, participant email, explicit workspace.client_domain column, or an org → domain join?
Is there an additional implicit filter (e.g. is_demo = false, tenant_id = key.tenant_id, deleted_at IS NULL) that could be eliminating everything?
What we suspect
One of these:

(A) Reports endpoint is filtering on a closed/reported status that no workspaces are actually in yet. (Plausible for reports — but workspaces directory is also empty, so probably not the only issue.)
(B) The ?domain= filter is matching against a column that's NULL/empty on existing workspaces. E.g. you added a client_domain column for the umbrella feature but never backfilled it on legacy workspaces, so every umbrella query returns zero.
(C) The umbrella key's allowed_domains doesn't include synozur.com. Easy to fix: add it.
(D) The endpoint has a tenant scoping clause that excludes Synozur's workspaces from the umbrella query (e.g. an internal-tenant guard).
(B) is the most likely cause given that every domain we tried returned zero, including ones that definitely have data.
